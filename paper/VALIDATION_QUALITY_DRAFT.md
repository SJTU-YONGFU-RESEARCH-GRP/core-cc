# Validation and Quality

The validation of the CORE-CC dataset ensures the reliability, accuracy, and physical implementability of the provided Error Correction Codes (ECCs). This section details the three-stage verification process: Algorithmic Validation using Python golden references, Functional Hardware Verification via Verilator co-simulation, and Physical Implementation Feasibility through Yosys synthesis.

## 1. Algorithmic Validation (Python Golden Reference)

## 2. Functional Hardware Verification (Verilator Co-Simulation)

To ensure that the generated Verilog RTL is a bit-exact representation of the algorithmic models, every module was verified using Verilator co-simulation. This process involved compiling the Verilog into a C++ cycle-accurate model and stimulating it with randomized test vectors generated by the Python reference.

**Verification Summary**

The table below summarizes the verification campaign results. All 26 primary ECC modules passed their respective test suites, with zero failures across 576 total test vectors. The verification suite is highly efficient, completing the full regression in under 3 minutes.

| Metric | Value |
| :--- | :--- |
| Total modules scanned | 39 |
| Primary ECC modules | 26 |
| Modules Passed | 26 |
| Modules Failed | 0 |
| **Module Pass Rate** | **100%** |
| Total test vectors executed | 576 |
| Total tests passed | 576 |
| **Test Pass Rate** | **100%** |
| Total runtime (s) | 136.27 |
| Average runtime per module (s) | 5.24 |

The 100% pass rate confirms the logic consistency between the software models and the hardware implementations, validating the automatic code generation pipeline.
